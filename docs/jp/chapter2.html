基本的な使用方法

インストール

実行環境・必要なパッケージ

AIPHADの実行環境・必要なパッケージは以下の通りです．

  • Python >= 3.6

  • numpy == 1.18.5

  • matplotlib == 3.2.2

  • scipy == 1.5.0

  • scikit-learn == 0.23.1

ダウンロード・インストール

  • PyPI からのインストール（推奨）

    $ pip3 install aiphad

    • NumPy などの依存パッケージも同時にインストールされます．

    • --user オプションを追加するとユーザのホームディレクトリ以下にインストールされます．

      $ pip3 install --user aiphad

  • ソースコードからのインストール（開発者向け）

    1. 本体のダウンロード

      ソースファイルをダウンロードするか，以下のように github レポジトリをクローンしてください．

      $ git clone https://github.com/tsudalab/aiphad

    2. pip を 19.0 以上に更新

      $ pip3 install -U pip

- ここで ``pip3`` が入っていない場合には ``python3 -m ensurepip`` でインストール可能です

    3. インストール

      $ cd aiphad
$ pip3 install --user ./

アンインストール

  1. 以下のコマンドを実行します．

    $ pip uninstall aiphad

AIPHADの基本構造

AIPHADは以下のような構成になっています(第3階層まで表示)．:

|--aiphad
|    |--aiphad.pdc_sampler
|    |    |--fit
|    |    |--us
|    |    |--cross_selection
|    |    |--make_data
|    |    |--make_label
|    |    |--preprocessing
|    |    |--calculate
|    |    |--postprocessing
|    |    |--us_*

各モジュールは以下のような構成で作成されています．

  • aiphad.pdc_sampler.fit : ラベル推定計算モジュール

  • aiphad.pdc_sampler.us : 不確実性スコア計算モジュール

  • aiphad.pdc_sampler.cross_selection : １変数を固定した場合の計算モジュール

  • aiphad.pdc_sampler.make_data : データ読み込みモジュール

  • aiphad.pdc_sampler.make_label : ラベル作成モジュール

  • aiphad.pdc_sampler.preprocessing : 前処理モジュール

  • aiphad.pdc_sampler.calculate : ラベル伝搬手法計算モジュール

  • aiphad.pdc_sampler.postprocessing : 後処理モジュール

  • aiphad.pdc_sampler.us_* : 不確実性スコアの計算モジュール（不確実性スコアの定義に応じた関数であり， * にはEA等が入る）

各モジュールの詳細については API Reference を参考にしてください．

PDCにおける計算の流れ

PDCを用いることで，相図を書くための実験回数を減らすことができ，さらに未知の相も効率的に発見することができます．PDCでは，以下の手順により不確実性サンプリングを実行します(それぞれの詳細は チュートリアル および API Reference を参考にしてください)．

  1. 描きたい相図の空間を定義

相図を描きたい空間を用意し，それを離散的に分割します． 分割された点の数を:math:N とすると， \(N\) 個の実験条件に対して実験を全て行えば，詳細な相図が得られることになります． 離散化された位置ベクトルを \(\mathbf{x} \in \mathbb{R}^d\) とすると，描きたい相図の実験条件候補は \(\mathbf{\{ x_\it i \}_{i= \rm 1,\ldots ,N} }\) で表されることになります． このとき，相図を描きたい空間次元 \(d\) は任意となります．

この \(N\) 個の実験条件のうち， \(M\) 個の実験条件の実験が終わっており，その点における相の種類が同定できているとします． この段階で見つかっている相のそれぞれのデータ点には，数値インデックスをラベルとして付与します． 例えば，見つかっている相の種類が \(P\) だった場合，相の種類を表すインデックスとして， \(p = 1, \ldots ,P\) を用意し，各データのラベルとして付与します． このとき，まだ実験を行っていない実験条件に対しては， \(-1\) のラベルを付与します．

  1. 相図推定 (phase estimation)

まだ相が同定されていない相図中の各点に対して，すでに見つかっている相に対する所属確率 \(P(p|\mathbf{x})\) をラベル伝搬法により推定します．PDCでは以下２種類のラベル伝搬法から選択します．

  • LP : label propagation法による所属確率計算

  • LS : label spreading法による所属確率計算

LP ではすでに相が同定されている点の相情報は不変なのに対し， LS では，相情報が周囲の環境によって変化する点が特徴的です．

この確率を利用し，各点で確率が最大となる相を予想される相とすることで，推定相図を描くことができます．

  1. 不確実性スコア (uncertainty score)

ステップ2で出力された所属確率を用いて，不確実性スコア (uncertainty score)を計算します．このスコアが最も大きくなる点（相図中で最も不確かな点）を，次の実験条件候補点として提案します．PDCでは，不確実性スコアの定義として，以下の4種類のスコアが利用できます．

  • LC : least confidentによる不確実性スコア ( \(1-P(p_1|\mathbf{x})\) )

  • EA : entropy-based approachによる不確実性スコア ( \(1-[P(p_1|\mathbf{x})-P(p_2|\mathbf{x})]\) )

  • MS : margin samplingによる不確実性スコア ( \(- \sum_{p} P(p|\mathbf{x})\log {P(p|\mathbf{x})}\) )

  • RS : random samplingによる不確実性スコア

ただし， \(P(p_1|\mathbf{x})\) および \(P(p_2|\mathbf{x})\) は，点 \(\mathbf{x}\) において最大の確率をもつ相の所属確率と，2番めに高い確率をもつ所属確率をそれぞれ表します． このとき，複数の実験条件候補を提案することもできます．

  1. 実験 (experiment)

ステップ3で選定された不確実性スコアが最大となる点に対して実験を行い，相を同定します．そして，同定された相のインデックスをその点のラベルとして入力します．このとき，新しい相が見つかった場合，相を表すインデックスを1つ追加し，新しいデータ点にラベルをつけます．

これにより学習データが追加され，このデータを利用し，ステップ2に戻り，ステップ2～4を繰り返すことで相図を効率的に描くことができます． 以上の流れを図示したものが以下になります．
